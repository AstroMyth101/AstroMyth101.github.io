Flagship Project · Spring-Summer 2026

Soft Robotic Rehabilitation Glove Controller

A portable electropneumatic controller for soft robotic finger motion, built around embedded control, pneumatic actuation, flex-sensor feedback, calibration, serial telemetry, and bench testing.

Problem

Repeatable assisted finger motion requires control, feedback, and safe bench behavior.

A soft robotic glove needs controlled pneumatic inflation and vacuum-assisted extension while measuring finger position through flex sensors. The engineering challenge is not only moving the glove, but making the system observable, adjustable, documentable, and testable.

System Architecture

Embedded controller → drivers → pumps and valves → soft glove → flex-sensor feedback.

  • ESP32-based controller running C++ firmware.
  • Pressure and vacuum pumps driven through motor drivers.
  • Central solenoid valve for pressure/vacuum routing.
  • Finger-level valves for individual pneumatic channel control.
  • Flex sensors used for angle estimation and feedback.
  • Serial telemetry and logging hooks for testing and analysis.

Firmware Features

Control logic written for practical debugging, not lab fantasy.

  • Finite-state machine for controlled inflation, hold, vacuum, and fault states.
  • Angle-setpoint control using flex-sensor feedback.
  • Calibration routines for converting raw sensor readings into usable angle estimates.
  • Serial command and telemetry interface for bench operation.
  • Fault handling for sensor limits, timeout conditions, and test safety boundaries.
  • Expansion path toward ROS 2/micro-ROS, EMG, IMU, and real-time logging.

Engineering Value

The project demonstrates integration across mechanical, electrical, firmware, and documentation work.

This project is useful because it forces multiple engineering domains to cooperate: pneumatic plumbing, embedded firmware, sensors, motor/valve drivers, control logic, data logging, and documentation. That is where real mechatronics lives. The polite word is integration. The honest word is chaos management.

Limitations

Clear boundaries matter.

This is a research and bench-validation platform. It should not be presented as a clinically approved device, a patient-ready system, or a human-tested rehabilitation product. The current value is engineering development, test data, controller validation, and documentation.
